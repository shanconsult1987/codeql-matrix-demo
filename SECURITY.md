# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates.

| Version | Supported |
| ------- | --------- |
| 2.x.x   | ✅ |
| 1.x.x   | ⚠️ Critical fixes only |
| < 1.0   | ❌ No longer supported |

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

DO NOT open a public GitHub issue.

Instead, report the vulnerability using one of the following methods:

Email: security@yourcompany.com

Include the following details:

• Description of the vulnerability  
• Steps to reproduce  
• Impact assessment  
• Suggested fix (if known)

---

## Response Timeline

We aim to respond according to the following timeline:

| Time | Action |
|-----|-------|
| 24 hours | Initial acknowledgement |
| 72 hours | Investigation |
| 7 days | Status update |
| 30 days | Fix or mitigation |

---

## Disclosure Policy

Once the vulnerability is fixed:

1. A security advisory will be published
2. CVE may be requested
3. Patch release will be issued

---

## Security Best Practices

Users of this project should:

• Keep dependencies updated  
• Use supported versions only  
• Monitor GitHub security advisories
